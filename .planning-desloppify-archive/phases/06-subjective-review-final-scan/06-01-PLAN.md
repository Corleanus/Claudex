### WP-03: Run per-file subjective review (63 files)

Execute `desloppify review --prepare` to generate review context, then review each file across the 7 dimensions and produce a `findings.json` file for import.

**Workflow**:

1. Run `desloppify review --prepare` — generates query.json with file contents, dimension prompts, and context
2. For each file in query.json's `files[]` array:
   - Read the file content (provided in query.json)
   - Evaluate against all 7 dimensions using the prompts and calibration guidance
   - Score assessments (0-100 per dimension)
   - Record specific findings with line numbers, evidence, and suggestions
3. Write results to `findings.json` in the format specified by the system_prompt in query.json
4. Import via `desloppify review --import findings.json`

**Dimensions** (scored 0-100 each):
- `naming_quality` — Names communicate intent
- `error_consistency` — Consistent error handling within modules
- `abstraction_fitness` — Abstractions earn their complexity
- `logic_clarity` — Control flow is provably correct
- `ai_generated_debt` — No LLM-hallmark patterns
- `type_safety` — Type annotations match runtime behavior
- `contract_coherence` — Functions honor stated contracts

**Parallelization**: Files are independent — multiple agents can review different file batches simultaneously. Group by module:

| Batch | Module | Files | Agent |
|-------|--------|-------|-------|
| A | `src/db/` | 16 files | Agent 1 |
| B | `src/shared/` + `src/checkpoint/` | 11 + 4 files | Agent 2 |
| C | `src/hooks/` + `src/wrapper/` | 8 + 2 files | Agent 3 |
| D | `src/lib/` + `src/gsd/` + `src/hologram/` | 9 + 7 + 4 files | Agent 4 |

Each batch produces a `findings-{batch}.json` file, imported separately.
