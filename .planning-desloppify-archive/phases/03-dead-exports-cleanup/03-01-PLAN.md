### The noUnusedLocals Constraint

`tsconfig.json` sets `noUnusedLocals: true`, but the `"exclude": ["tests"]` means test files aren't checked. This creates a catch-22 for the 7 truly dead symbols:
- If we remove `export`, tsc complains about unused locals
- If we keep `export`, desloppify complains about dead exports

Resolution options for each:
- **Delete the declaration entirely** if no code references it at all (not even internally)
- **Prefix with underscore** (`_DatabaseError`) — tsc allows unused locals prefixed with `_` when `noUnusedLocals` is true... but TypeScript does NOT support this convention for `noUnusedLocals` (only for `noUnusedParameters`). So this won't work.
- **Keep exported + mark wontfix** if the symbol is part of a public API surface that callers *should* use but don't yet

### WP-02: Triage the 7 truly dead symbols

For each of these 7 symbols, determine whether to **delete** or **keep as wontfix**:

#### Group A: Error classes (`DatabaseError`, `HookError`, `ConfigError` in `src/shared/errors.ts`)

These are typed error classes that extend `ClaudexError`. They exist as part of a public error hierarchy but are never actually thrown or caught anywhere. However:
- They represent a clean, intentional API surface (error taxonomy)
- Future code may want to throw/catch these specific types
- Deleting them removes a designed API contract for no functional gain

**Decision**: Mark as `wontfix` with note "Intentional error taxonomy — available for future use, zero runtime cost."

#### Group B: Type-only interfaces (`HologramQuery`, `SchemaVersioned`, `CrossPhaseData`, `VersionedInput`)

These are TypeScript interfaces — zero runtime cost, purely type definitions. They describe data shapes that may be used by future code or external consumers:
- `HologramQuery`: describes query params for hologram sidecar
- `SchemaVersioned`: describes versioned schema contracts
- `CrossPhaseData`: describes cross-phase data passing
- `VersionedInput`: describes versioned hook input

**Decision**: Mark as `wontfix` with note "Type-only interface — zero runtime cost, part of designed type surface."

For all 7, run:
```
desloppify resolve <finding-id> --status wontfix --note "<rationale>"
```
